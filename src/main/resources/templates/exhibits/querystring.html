Query String Handoff

tagline

Why enterprises do this

App A needs to tell App B "the user picked Mission Pharmacy." The fastest thing that works: put it in the link. ?accountId=ACC-A&role=ADMIN and you're done — no backend, no shared store.

Why it hurts

  • Sensitive data lands in browser history, server logs, referrers, and chat messages.
  • Brittle, undocumented param names drift between apps.
  • Unencoded values (&, spaces) corrupt the whole query string.
  • A copied URL replays old context forever — no expiry.

Scripted walkthrough

  1. BAD: Generate handoff URL and read it — the account id, role, and a fake token are right there in the link (and in the log, like a real access log).
  2. BAD: Tick tricky account name, regenerate, then Open target app: the unencoded & corrupts the parsed context.
  3. BAD: Replay the copied URL later — stale context is happily accepted again. Nothing expires.
  4. BETTER: Create server-side handoff (URL is just an opaque id), Open target app (works once), then Replay → a clear expired/used error.
